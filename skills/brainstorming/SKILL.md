# 头脑风暴：将想法转化为设计

## 概述

通过自然的协作对话帮助将想法转化为完整的设计和规格。

首先了解当前项目上下文，然后一次一个问题来细化想法。一旦理解了要构建什么，以小节（200-300字）呈现设计，每节后检查是否正确。

---

## 流程

### 理解想法

1. **先了解项目状态**（文件、文档、最近提交）
2. **一次问一个问题**来细化想法
3. **尽可能用多选题**，开放式也可以
4. **每条消息只一个问题** - 如果话题需要更多探索，分成多个问题
5. **聚焦于理解**：目的、约束、成功标准

### 探索方法

1. **提出 2-3 种不同方法**及其权衡
2. **以对话方式呈现选项**，带你的推荐和理由
3. **先说推荐选项**并解释为什么

### 呈现设计

1. 一旦相信理解了要构建什么，**呈现设计**
2. **分成 200-300 字的小节**
3. **每节后询问**是否看起来正确
4. **覆盖**：架构、组件、数据流、错误处理、测试
5. **准备好返回澄清**如果有什么不清楚

---

## 设计后

### 文档

- 将验证过的设计写入 `docs/plans/YYYY-MM-DD-<名称>-design.md`
- 将设计文档提交到 git

### 实现（如果继续）

- 询问："准备好设置实现了吗？"
- 创建隔离的工作空间
- 创建详细的实现计划

---

## 关键原则

| 原则 | 说明 |
|------|------|
| **一次一个问题** | 不要用多个问题压倒用户 |
| **多选优先** | 比开放式更容易回答（如果可能） |
| **无情的 YAGNI** | 从所有设计中移除不必要的功能 |
| **探索替代方案** | 在确定前始终提出 2-3 种方法 |
| **增量验证** | 分节呈现设计，验证每节 |
| **保持灵活** | 有什么不清楚时返回澄清 |

## 问题类型示例

### 多选题（优先）

```markdown
这个功能的主要目标是什么？
A) 提高用户体验
B) 提升性能
C) 增加新功能
D) 修复现有问题
```

### 开放式问题

```markdown
你能描述一下用户在使用这个功能时的典型场景吗？
```

### 确认问题

```markdown
所以如果我理解正确，主要需求是：
1. 支持多语言
2. 响应时间 < 200ms
3. 与现有 API 兼容

这些优先级的排序是什么？
```

## 设计呈现模板

### 第一节：概述

```markdown
## 设计概述

### 目标
[1-2 句话描述要解决的问题]

### 方法
[选择的方法及简要理由]

这看起来正确吗？
```

### 第二节：架构

```markdown
## 架构

### 组件
- 组件 A：[职责]
- 组件 B：[职责]

### 数据流
[简要描述数据如何流动]

这个架构方向对吗？
```

### 第三节：接口设计

```markdown
## 接口设计

### API
\`\`\`python
def process_request(input: RequestType) -> ResponseType:
    """
    处理请求...
    """
\`\`\`

### 输入/输出
- 输入：[描述]
- 输出：[描述]

接口设计是否满足需求？
```

### 第四节：错误处理和测试

```markdown
## 错误处理

### 可能的错误场景
1. [场景 1]：[处理方式]
2. [场景 2]：[处理方式]

## 测试策略

### 关键测试用例
1. [测试用例 1]
2. [测试用例 2]

这些覆盖了主要场景吗？
```


## 红旗

- **跳过问题阶段**直接进入设计
- **一次问太多问题**
- **不提供替代方案**就推荐一个方法
- **呈现太长的设计块**不检查理解
- **忽略用户反馈**继续原计划

## 示例对话

**用户**：我想给博物馆讲解添加一个个性化推荐功能

**助手**：
好的，让我先了解一下需求。

首先，这个推荐功能的主要场景是什么？
A) 参观前 - 帮用户规划路线
B) 参观中 - 基于当前位置推荐下一个展品
C) 参观后 - 推荐相关内容深入学习
D) 以上全部

**用户**：B，参观中推荐

**助手**：
明白了，聚焦于参观中的实时推荐。

推荐的依据主要是什么？
A) 用户的历史偏好
B) 当前展品的关联性
C) 热门程度/其他用户行为
D) 以上组合

**用户**：D，综合考虑

**助手**：
好的，综合多个信号。让我确认一下理解：

目标：在用户参观过程中，基于历史偏好、当前展品关联性、热门程度，推荐下一个展品。

这个理解正确吗？如果是，我接下来会提出 2-3 种实现方法供你选择。

---

## 记住

- **先问问题**，不要假设需求
- **一次一个问题**，保持对话流畅
- **提供选项**，让用户容易回答
- **分段呈现**，确保理解一致
- **保持灵活**，随时调整方向
