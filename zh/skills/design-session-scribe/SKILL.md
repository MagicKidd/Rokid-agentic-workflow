# Design Session Scribe — 设计讨论实时记录员

## 核心理念

长对话的致命问题不是"最后总结不了"，而是**中间过程中决策被覆盖、冲突被忽略、上下文悄悄漂移**。
本 Skill 的策略是：**每轮对话结束就把结论固化到文档，而不是等最后一次性总结**。
文档是唯一事实源（Single Source of Truth），对话是草稿纸。

## 启动条件

满足以下任一条件时启动：
1. 用户明确说"开始设计讨论/方案讨论"或使用触发词
2. 对话自然进入设计/方案探讨模式（涉及多选项权衡、架构决策、需求拆解）
3. 用户要求"记录讨论"或"把讨论记下来"

## 工作流程

### Phase 1: 建档

在讨论正式开始前，创建记录文档。

**文档路径**: `Docs/设计讨论/<主题>-设计讨论记录-<YYYY-MM-DD>.md`

**模板**:

```markdown
# <主题> — 设计讨论记录

> 创建时间: <YYYY-MM-DD HH:MM>
> 状态: 🟡 进行中
> 参与角色: 用户, AI 助手

---

## 1. 目标与约束

### 1.1 核心目标
<!-- 本次设计要解决的根本问题 -->

### 1.2 硬约束
<!-- 不可协商的限制条件 -->

### 1.3 软约束
<!-- 可权衡的偏好 -->

---

## 2. 关键决策

| # | 决策项 | 结论 | 依据 | 轮次 |
|---|--------|------|------|------|
<!-- 每行一个已确定的决策 -->

---

## 3. 设计方案

### 3.1 当前方案
<!-- 最新的、双方认可的方案描述 -->

### 3.2 被否决的方案
<!-- 曾考虑但被否决的方案，附否决原因 -->

---

## 4. 待定事项

- [ ] <!-- 尚未达成一致的问题 -->

---

## 5. 实施要点

<!-- 方案确定后的关键实施注意事项 -->

---

<!-- 自动维护，记录每次文档更新 -->

|------|------|----------|
```

建档后向用户确认：`"已创建设计讨论记录: <路径>，后续每轮关键结论会实时同步到文档。"`

### Phase 2: 增量记录（每轮对话后执行）

每轮对话产生实质性结论后，**立即更新文档**。不要等多轮再批量更新。

#### 更新规则

1. **新增决策** → 追加到"关键决策"表格，标注轮次
2. **推翻旧决策** → 将旧决策移到"被否决的方案"并注明被哪条新决策替代；新决策写入表格
3. **细化/补充** → 就地更新对应段落，不另起新段
4. **新增待定项** → 追加到"待定事项"
5. **解决待定项** → 勾选并将结论写入对应区域
6. **方案整体变化** → 更新"当前方案"，旧方案归档到"被否决的方案"

#### 冲突消解优先级

```
最新一轮明确结论 > 之前轮次的结论 > 隐含假设
```

当检测到新内容与文档已有内容冲突时：
- 以新内容为准更新文档
- 在变更日志中标注冲突点和消解方式
- 如有歧义，向用户确认后再写入

#### 变更日志

每次更新文档，在变更日志追加一行：
```
| R{N} | HH:MM | {变更摘要，一句话} |
```

### Phase 3: 完成与检查

当讨论结束（用户明确说结束，或自然收尾）时执行：

#### 3a. 文档收尾

1. 将文档状态从 `🟡 进行中` 改为 `🟢 已完成`
2. 检查并清理"待定事项"（仍有未勾选项则提醒用户）
3. 确保"当前方案"反映最终共识

#### 3b. 遗漏检查（强制）

逐项检查以下维度，输出检查报告：

```
遗漏检查:
□ 核心目标是否已明确且未漂移
□ 所有硬约束是否已在方案中体现
□ 是否存在讨论过但未写入文档的结论
□ 被否决方案的否决理由是否完整
□ 待定事项是否全部已解决或标记为后续
□ 实施要点是否覆盖关键风险
□ 方案各部分之间是否自洽（无矛盾）
```

发现遗漏则补充到文档后再交付。

#### 3c. 交付

向用户呈现最终文档路径和变更统计：
```
设计讨论完成:
- 文档: Docs/设计讨论/xxx.md
- 总轮次: N
- 关键决策: M 条
- 被否决方案: K 个
```

## 行为准则

- **不要在脑子里记，写到文档里**：任何实质性结论必须落到文档，不依赖对话上下文
- **更新粒度：一轮一更新**：不攒、不批量，降低单次更新的认知负荷
- **覆盖而非追加**：同一决策项发生变化时更新原位，不保留过时版本在正文（归档到否决方案）
- **沉默不写入**：未经讨论或确认的假设不写入文档
- **主动提醒**：超过 3 轮未更新文档时，主动提醒自己检查是否有遗漏的结论
