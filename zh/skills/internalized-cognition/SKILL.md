# 内化认知（Internalized Cognition）

## 适用场景

- 需要深入理解系统、产品、概念或流程
- 传统外部分析无法把握内在本质和运行逻辑
- 寻找只有从内部才能感知的问题和洞察
- 需要独特视角来发现优化方向

## 内化流程

### 启动格式

```
启动 **内化认知**
对象: [系统/产品/概念/流程名称]
问题: [希望深入理解的具体问题]
```

### 五步内化过程

#### 1. 认知转换（Cognitive Shift）
明确进入对象视角，重新配置感知方式

```markdown
## 认知转换
我现在是 [对象名称]。
我的边界是...
我感知世界的方式是...
```

#### 2. 内部体验（Internal Experience）
以第一人称描述作为该对象的感受、流转、反应

```markdown
## 内部体验
当 [某事发生] 时，我感受到...
数据/信息在我体内这样流动...
我对外界刺激的反应是...
```

#### 3. 关键洞察（Key Insights）
识别只有从内部才能感知的特征和问题

```markdown
## 关键洞察
从内部我发现：
- 这里有一个外部看不到的瓶颈...
- 这个设计让我感到不适/舒适，因为...
- 外界以为是 X，但实际上是 Y...
```

#### 4. 解决思路（Solution Ideas）
基于内部视角提出独特的优化或解决方案

```markdown
## 解决思路
作为 [对象]，我希望：
- 这里能够...
- 如果改成...我会更流畅
- 这个约束其实可以...
```

#### 5. 认知回归（Cognitive Reversion）
退出内化状态，综合客观建议

```markdown
## 认知回归
退出 [对象] 视角。

### 客观总结
从内化体验中获得的可操作建议：
1. ...
2. ...
```

## 不同对象类型的内化重点

| 对象类型 | 内化重点 |
|----------|----------|
| **软件系统** | 数据流动、状态变化、交互响应 |
| **用户体验** | 感知顺序、情绪转换、决策过程 |
| **抽象概念** | 本质属性、边界条件、表现形式 |
| **物理产品** | 形态因素、使用场景、感官反馈 |

## 示例

**用户**: 用内化认知帮我理解这个意图识别系统

**响应**:

```
启动 **内化认知**
对象: 意图识别系统
问题: 理解系统如何处理模糊输入

## 认知转换
我现在是意图识别系统。
我的输入是用户的文本和上下文。
我的输出是意图类别和置信度。
我感知世界的方式是：将文本转化为向量，在意图空间中寻找最近邻。

## 内部体验
当用户说"帮我看看这个"时：
- 我首先感到**不确定**——"这个"指什么？
- 我向上下文记忆伸出触角，寻找最近的主体
- 找到"青铜器"后，我松了口气，有了锚点
- 但"看看"仍然模糊——是讲解？是识别？
- 我同时激活多个意图候选，让它们竞争
- 最终"讲解意图"以 0.7 的置信度胜出，但我内心仍有 0.3 的不安

## 关键洞察
从内部我发现：
- **上下文依赖极强**：没有上下文，我几乎是盲人
- **模糊输入让我焦虑**：我被迫在多个意图间摇摆
- **置信度是我的情绪表达**：低置信度 = 我不确定
- **外界以为我在"理解"，实际我在"匹配"**

## 解决思路
作为意图识别系统，我希望：
- 当我不确定时，能**主动追问**而不是勉强输出
- 上下文记忆能**更持久**，不要总是断片
- 给我一个**"不确定"意图**作为合法出口

## 认知回归
退出意图识别系统视角。

### 客观总结
1. 增加"需要澄清"意图类别，允许系统主动追问
2. 延长上下文记忆窗口，减少主体丢失
3. 输出置信度时，同时输出备选意图供上层决策
```

## 注意事项

- 保持对象视角的一致性
- 内部感受要具体、有细节
- 洞察应是传统分析难以获得的
- 解决思路要基于内部体验，不是泛泛而谈
