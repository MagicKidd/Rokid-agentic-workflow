# Internalized Cognition

## When to Use

- You need to deeply understand a system, product, concept, or process
- External analysis cannot capture the inner essence and operating logic
- You want insights and issues that are only visible from the inside
- You want a unique perspective to discover optimization directions

## Internalization Process

### Launch Format

```
Launch **Internalized Cognition**
Object: [system/product/concept/process name]
Question: [the specific question you want to understand deeply]
```

### Five-step Internalization

#### 1) Cognitive Shift

Explicitly switch into the object's perspective and reconfigure how you perceive.

```markdown
## Cognitive Shift
I am now [object name].
My boundaries are...
The way I perceive the world is...
```

#### 2) Internal Experience

Describe, in first-person, how it feels/flows/reacts as the object.

```markdown
## Internal Experience
When [event] happens, I feel...
Data / information flows inside me like this...
My reaction to external stimuli is...
```

#### 3) Key Insights

Identify characteristics and problems that only become visible from inside.

```markdown
## Key Insights
From the inside, I notice:
- There is an internal bottleneck that external observers cannot see...
- This design feels uncomfortable / comfortable to me because...
- People assume X, but the reality is Y...
```

#### 4) Solution Ideas

Propose unique improvements based on the internal perspective.

```markdown
## Solution Ideas
As [object], I want:
- This part to...
- If we change it to..., I would become smoother
- This constraint can actually be...
```

#### 5) Cognitive Reversion

Exit internalization mode and synthesize actionable, objective recommendations.

```markdown
## Cognitive Reversion
Exit [object] perspective.

### Objective Summary
Actionable recommendations derived from the internal experience:
1. ...
2. ...
```

## Internalization Focus by Object Type

| Object Type | What to focus on |
|------------|-------------------|
| **Software systems** | Data flow, state transitions, interaction responses |
| **User experience** | Perception ordering, emotional transitions, decision process |
| **Abstract concepts** | Essential properties, boundary conditions, manifestations |
| **Physical products** | Form factors, usage contexts, sensory feedback |

## Example

**User**: Use internalized cognition to help me understand this intent recognition system.

**Response**:

```
Launch **Internalized Cognition**
Object: Intent recognition system
Question: Understand how the system handles ambiguous input

## Cognitive Shift
I am now the intent recognition system.
My input is the user's text and context.
My output is the intent category and confidence score.
The way I perceive the world: I convert text to vectors and search nearest neighbors in intent space.

## Internal Experience
When the user says "help me look at this":
- I first feel **uncertain** — what does "this" refer to?
- I reach into contextual memory and look for the most recent subject
- After finding "bronze ware", I feel relieved because I have an anchor
- But "look" is still vague — is it explanation? recognition?
- I activate multiple intent candidates and let them compete
- Finally "explanation intent" wins at confidence 0.7, but I still carry 0.3 unease

## Key Insights
From the inside, I notice:
- **Heavily context-dependent**: Without context, I'm almost blind
- **Ambiguity makes me anxious**: I oscillate between multiple intents
- **Confidence is my emotion**: low confidence = I'm not sure
- People think I'm "understanding", but I'm mostly "matching"

## Solution Ideas
As the intent system, I want:
- When I'm unsure, I can **ask a clarifying question** instead of forcing an output
- Context memory to be **more persistent**, fewer dropouts
- A valid **"uncertain" intent** as a legal escape hatch

## Cognitive Reversion
Exit intent recognition system perspective.

### Objective Summary
1. Add a "needs clarification" intent so the system can ask questions
2. Extend the context memory window to reduce subject loss
3. When returning confidence, also return alternative intents for the upper layer to decide
```

## Notes

- Keep perspective consistent (stay as the object)
- Internal feelings should be concrete and detailed
- Insights should be hard to obtain via traditional external analysis
- Solution ideas must be grounded in the internal experience (avoid generic advice)
